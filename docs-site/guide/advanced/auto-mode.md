# 自动模式

自动模式允许你配置规则，控制 AI 在执行操作时的自动审批行为，减少人工确认的打断。

在 **设置 → 自动模式** 中配置。

## 工作原理

当 AI 请求执行某个操作（如编辑文件、运行命令）时，自动模式会根据你配置的 **允许/拒绝规则** 自动判断是否批准：

- 匹配 **允许规则** → 自动批准，无需用户手动确认
- 匹配 **拒绝规则** → 自动拒绝
- 不匹配任何规则 → 按当前 Permission 模式处理

## 规则类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 允许规则 | 匹配的操作自动批准 | 允许读取 `src/` 目录下的文件 |
| 拒绝规则 | 匹配的操作自动拒绝 | 拒绝修改 `package.json` |

## 编辑模式

### 列表模式

可视化界面，逐条添加规则：

1. 选择规则类型（允许/拒绝）
2. 定义匹配条件（操作类型、目标路径等）
3. 保存生效

### JSON 高级模式

直接编辑 JSON 配置，适合批量管理和复杂规则：

```json
{
  "allow": [
    { "tool": "Read", "path": "src/**" },
    { "tool": "Grep", "path": "**" }
  ],
  "deny": [
    { "tool": "Edit", "path": "package.json" },
    { "tool": "Bash", "command": "rm -rf *" }
  ]
}
```

## 默认规则

系统内置了默认规则集，可以在面板中查看。默认规则覆盖了常见的安全操作：

- 自动允许：读取文件、搜索文件等只读操作
- 自动拒绝：删除系统文件、执行危险命令等

## 最佳实践

| 建议 | 说明 |
|------|------|
| 从保守开始 | 先使用 Default 权限模式，逐步添加自动规则 |
| 保护关键文件 | 将 `package.json`、`.env`、配置文件加入拒绝规则 |
| 使用通配符 | 用 `src/**` 匹配整个目录而非逐文件配置 |
| 定期审查 | 定期检查规则列表，清理不再需要的规则 |
| 备份配置 | 修改前复制 JSON 配置备份，避免误操作 |
