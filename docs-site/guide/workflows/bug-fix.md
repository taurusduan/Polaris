# 实战：用 Polaris 修复一个 Bug

本文演示如何使用 Polaris 从发现 Bug 到提交修复的完整流程。

## 场景描述

假设你发现项目中的某个按钮点击后没有反应，需要排查和修复这个问题。

## Step 1：创建工作区

如果你还没有打开项目工作区：

1. 点击顶栏的 **工作区选择器**
2. 点击「创建新工作区」
3. 输入工作区名称，选择项目文件夹路径
4. 确认创建并切换

## Step 2：向 AI 描述问题

在 AI 对话面板中描述你遇到的问题：

```
我在页面上点击「提交」按钮后没有任何反应，
按钮位于 src/components/SubmitForm.tsx 中，
请帮我排查这个 Bug。
```

AI 会：
1. 读取相关文件，分析代码逻辑
2. 展示工具调用卡片（显示读取了哪些文件）
3. 给出问题分析和修复建议

## Step 3：查看工具调用过程

AI 在分析过程中会执行多个工具调用，你可以：

- 点击工具调用卡片展开查看每个步骤的详情
- 查看 AI 读取了哪些文件、搜索了哪些关键词
- 理解 AI 的分析思路

## Step 4：审批 AI 的修复方案

如果 AI 进入计划模式：

1. 查看计划中包含的阶段和操作步骤
2. 点击 **批准** 让 AI 执行修复
3. 或者点击 **拒绝** 并补充修改建议

如果 AI 需要修改文件，可能弹出权限请求：

1. 查看 AI 要执行的操作类型和目标
2. 点击 **批准** 或 **拒绝**

## Step 5：查看文件变更

AI 修改文件后：

1. 打开左侧 **Git 面板** 📄（点击 ActivityBar 的 GitPullRequest 图标）
2. 切换到 **变更** 标签页
3. 查看所有被修改的文件和变更类型
4. 点击文件名查看差异对比视图

## Step 6：验证修复

1. 在 **终端** 面板中运行项目测试：`pnpm test`
2. 或者手动在浏览器中测试功能
3. 如果测试通过，进入下一步提交代码

## Step 7：提交修复

回到 Git 面板的变更标签页：

1. 点击 **暂存全部** 将所有变更添加到暂存区
2. 点击 **AI 生成提交信息** 按钮，让 AI 自动生成 commit message
3. 检查并编辑提交信息（如 `fix: 修复提交按钮点击无响应的问题`）
4. 点击 **提交**

## Step 8：推送到远程

1. 切换到 Git 面板的 **远程** 标签页
2. 点击 **推送** 将本地提交推送到远程仓库

## 总结

| 步骤 | 使用的功能 |
|------|-----------|
| 打开项目 | 工作区管理 |
| 描述问题 | AI 对话 |
| 分析代码 | 工具调用可视化 |
| 审批修复 | 计划模式 / 权限请求 |
| 查看变更 | Git 面板 / 差异对比 |
| 运行测试 | 终端 |
| 提交代码 | Git 提交 / AI 生成提交信息 |
| 推送远程 | Git 远程管理 |
