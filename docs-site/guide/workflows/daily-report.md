# 实战：定时任务生成日报

本文演示如何配置定时任务，让 AI 每天自动生成项目日报。

## Step 1：打开定时任务面板

通过 ActivityBar 的 **Clock** ⏰ 图标打开定时任务面板。

确认顶部调度器状态为「运行中」。如果显示「已停止」，点击启动按钮。

## Step 2：创建日报任务

点击面板右上角的「新建」按钮：

### 基本配置

| 字段 | 值 |
|------|-----|
| 任务名称 | 每日项目日报 |
| 描述 | 自动分析 Git 提交记录和代码变更，生成项目日报 |
| 任务模式 | 简单模式 |
| 任务分类 | 自定义 |

### 触发配置

选择 **Cron 表达式**：

| 目标 | Cron 表达式 |
|------|------------|
| 每天早上 9 点 | `0 9 * * *` |
| 工作日早上 9 点 | `0 9 * * 1-5` |
| 每天下午 6 点 | `0 18 * * *` |

### 提示词配置

在提示词区域输入：

```
请基于今天的项目活动生成一份日报，包含以下内容：

## 今日完成
- 列出今天的 Git 提交记录（git log --since="today"）
- 总结每个提交做了什么

## 代码变更统计
- 修改了哪些文件
- 新增/删除/修改的代码行数

## 进行中的工作
- 当前未提交的变更概览

## 明日计划建议
- 基于当前进度，建议明天的工作重点

请使用 Markdown 格式输出。
```

### AI 引擎

选择你想使用的模型提供商和具体模型。

## Step 3：测试运行

创建任务后，先手动触发一次测试：

1. 在任务卡片上找到手动触发按钮
2. 点击执行，观察执行过程
3. 查看执行日志，确认输出格式和内容符合预期
4. 如果不满意，调整提示词后重新测试

## Step 4：查看执行日志

每次任务执行后，执行日志自动记录：

1. 点击任务卡片上的日志按钮打开 **执行日志抽屉**
2. 查看历次执行的时间、状态和输出内容
3. 可拖拽调整日志面板高度

## Step 5：使用协议模式（进阶）

如果简单模式的日报格式不够灵活，可以切换到协议模式：

1. 先在 **模板管理** 中创建协议模板
2. 定义参数化字段（如日期范围、输出格式、关注重点等）
3. 创建任务时选择协议模式，关联模板
4. 填写模板参数

协议模式的优势：
- 支持参数化配置，不用每次修改提示词
- 可以定义明确的任务目标
- 适合标准化的周期性任务

## 管理日常任务

### 暂停/恢复

- 临时不需要日报时，可以禁用任务（不需要删除）
- 需要时重新启用即可

### 调整时间

- 修改 Cron 表达式调整执行时间
- 支持「完成后间隔」模式：上次执行完成后等待指定时间再执行

### 多任务协作

可以创建多个定时任务配合使用：

| 任务 | Cron | 用途 |
|------|------|------|
| 每日日报 | `0 9 * * 1-5` | 工作日早上生成日报 |
| 代码审查 | `0 18 * * 1-5` | 工作日晚间自动审查当日提交 |
| 依赖检查 | `0 10 * * 1` | 每周一检查依赖更新 |
