# 开发者面板

开发者面板是面向高级用户和应用开发者的调试工具，提供应用内部 AI 事件的实时可视化和过滤。

通过 ActivityBar 的 **Code2** 图标打开开发者面板。

## 适用场景

| 场景 | 说明 |
|------|------|
| 调试 AI 行为 | 查看 AI 请求和响应的完整事件流 |
| 排查工具调用 | 定位工具调用失败、权限拒绝等问题的原因 |
| 监控会话状态 | 跟踪会话创建、结束、异常等生命周期事件 |
| 开发插件/集成 | 开发 MCP 服务器或平台集成时观察事件交互 |

## 事件过滤

面板顶部提供多维度过滤：

| 过滤维度 | 说明 |
|----------|------|
| 全部事件 | 显示所有事件，不做任何过滤 |
| 按 Task ID | 仅显示指定任务 ID 的事件（定时任务场景） |
| 按 Session ID | 仅显示指定会话 ID 的事件 |
| 全部类型 | 按事件类型过滤，快速定位特定行为 |

## 事件类型

AI 运行时会产生以下类型的事件：

| 事件类型 | 说明 |
|----------|------|
| `session_start` / `session_end` | 会话创建和结束 |
| `token` | 流式 Token 输出（AI 回复内容） |
| `tool_call_start` | 工具调用开始（AI 发起操作） |
| `tool_call_end` | 工具调用结束（包含执行结果） |
| `progress` | 进度更新（长时间操作） |
| `result` | 最终结果返回 |
| `error` | 错误事件（连接失败、超时等） |
| `permission_request` | 权限请求事件 |

## 事件列表

事件列表实时滚动展示：

| 功能 | 说明 |
|------|------|
| 实时流显示 | 新事件自动追加到列表底部 |
| 自动滚动 | 可开关，开启时自动滚动到最新事件 |
| 事件计数 | 显示当前过滤后的事件数量和最大显示数 |
| 复制 JSON | 每条事件支持一键复制完整 JSON 数据 |
| 清空事件 | 清除当前所有显示的事件 |

## 典型调试流程

### 场景一：AI 工具调用失败

1. 在事件类型中过滤 `tool_call_end`
2. 查看失败的工具调用事件
3. 复制 JSON 数据，检查 `error` 字段
4. 根据错误信息调整配置或修复问题

### 场景二：定时任务执行异常

1. 按 Task ID 过滤目标任务
2. 查看该任务的完整事件流
3. 关注 `session_start` → `tool_call_start/end` → `result` 的完整链路
4. 确认定时任务的执行是否正常完成

### 场景三：权限被意外拒绝

1. 过滤 `permission_request` 事件类型
2. 查看被拒绝的权限请求详情
3. 检查 Auto-Mode 规则配置（**设置 → 自动模式**）
4. 调整允许/拒绝规则后重试

## 性能说明

- 开发者面板仅读取事件流，不会影响 AI 对话性能
- 最大显示事件数可在面板中配置，防止内存占用过高
- 建议在排查问题时开启，日常使用时可关闭以减少界面干扰
