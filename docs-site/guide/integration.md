# 平台集成

Polaris 支持对接外部平台，通过机器人接收和发送消息。

通过 ActivityBar 的 **Bot** 图标打开机器人管理面板。

## QQ Bot

在 **设置 → QQ Bot** 中配置。

### 实例管理

支持添加多个机器人实例，每个实例包含：

| 配置 | 说明 |
|------|------|
| 实例名称 | 例如「生产机器人」 |
| App ID | QQ 开放平台应用的 App ID |
| Client Secret | QQ 开放平台应用的 Client Secret |
| 沙箱环境 | 开启后使用沙箱模式，用于测试 |
| 自动连接 | 启动时自动连接 WebSocket |
| 显示模式 | 聊天模式 / 分离模式 / 两者都有 |

### 连接状态

| 状态 | 说明 |
|------|------|
| 未连接 | 初始状态 |
| 连接中... | 正在建立 WebSocket 连接 |
| 鉴权中... | 正在进行身份验证 |
| 已就绪 | 连接成功，可收发消息 |
| 连接失败 | 连接出错，显示错误原因 |
| 重连中... | 连接断开后自动重连 |

### 错误类型

| 错误 | 原因 |
|------|------|
| Get token failed | App ID 或 Client Secret 不正确 |
| Get gateway failed | 无法获取 WebSocket 网关地址 |
| Connect timeout | 连接超时 |
| Auth failed | 鉴权失败 |
| WebSocket error | WebSocket 协议错误 |
| Connection closed | 连接被关闭 |

### 注意事项

- 保存设置后才能连接
- 连接会自动保存配置
- 同一时间仅一个实例可处于连接状态
- 建议先用沙箱环境测试

## 飞书

在 **设置 → 飞书** 中配置飞书机器人集成。

### 实例管理

支持添加多个飞书机器人实例，每个实例包含：

| 配置 | 说明 |
|------|------|
| 实例名称 | 例如「团队飞书机器人」 |
| App ID | 飞书开放平台应用的 App ID |
| App Secret | 飞书开放平台应用的 App Secret |
| Verification Token | 事件订阅验证令牌 |
| Encrypt Key | 事件加密密钥 |
| 显示模式 | 聊天模式 / 分离模式 / 两者都有 |
| 自动连接 | 启动时自动建立连接 |
| 工作目录 | 机器人关联的工作区路径 |

### 配置步骤

1. 前往 [飞书开放平台](https://open.feishu.cn/) 创建应用
2. 获取 App ID 和 App Secret
3. 配置事件订阅，获取 Verification Token 和 Encrypt Key
4. 在 Polaris **设置 → 飞书** 中填入配置
5. 保存设置并连接

### 连接状态

飞书机器人的连接状态与 QQ Bot 相同，支持自动重连。

### 注意事项

- 保存设置后才能连接
- 同一时间仅一个飞书实例可处于连接状态
- 飞书和 QQ Bot 可以同时使用不同的平台实例

## 平台面板

通过 ActivityBar 的 **Bot** 🤖 图标打开机器人管理面板，面板内通过标签页切换 QQ Bot 和飞书：

- 查看所有实例的连接状态
- 快速切换活跃实例
- 管理实例的增删改查
- 查看实时消息流
