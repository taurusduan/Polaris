# 常见问题

## 安装与连接

### 启动后显示「连接失败」怎么办？

1. 确认已安装 Claude CLI：`npm install -g @anthropic-ai/claude-code`
2. 在连接页面点击「重新检测」
3. 如果仍检测不到，点击「设置 Claude 路径」手动输入完整路径
4. Windows 用户检查 `C:\Users\<用户名>\AppData\Roaming\npm\claude.cmd` 是否存在
5. 确认 CLI 已完成首次登录认证（在终端中运行 `claude` 命令）

### 支持哪些操作系统？

- Windows 10 / 11（推荐，稳定支持）
- macOS 12+（实验性支持）

### 需要 API Key 吗？

Polaris 通过 Claude CLI 连接，使用 Claude CLI 的认证方式。首次使用需在终端运行 `claude` 命令完成登录。

### 如何更新 Claude CLI？

```bash
npm update -g @anthropic-ai/claude-code
```

更新后在 Polaris 的 **设置 → AI 引擎** 中查看版本号确认。

## AI 对话

### 如何切换 AI 模型？

在聊天面板底部状态栏中点击 **Model** 下拉框，选择 Sonnet / Opus / Haiku。不同会话可以使用不同的模型。

### 对话太长导致变慢？

Polaris 会自动压缩较早的对话内容以节省内存。关键信息会保留，但详细的代码块等可能被压缩为摘要。

你也可以：
- 开启新会话，避免单个会话过长
- 定期清理不需要的历史会话

### 如何查看工具调用的详情？

AI 执行操作时会显示工具调用卡片，点击「展开」可查看详细信息，包括文件差异、命令输出等。

### 对话中的思考块和工具调用太多？

当连续的思考块和工具调用超过 5 个时，Polaris 会自动折叠为一组。你可以：
- 点击折叠组展开查看全部详情
- 使用全局展开/收起按钮批量操作

### 如何让 AI 自动执行而不需要每次确认？

在状态栏中将 **Permission** 切换为 **Auto** 或 **AcceptEdits** 模式。更细粒度的控制可以在 **设置 → 自动模式** 中配置允许/拒绝规则。

### 输入框中如何引用文件？

- 输入 `@` 会弹出文件路径补全
- 使用 `@/path/to/file` 引用当前工作区的文件
- 使用 `@{工作区名}:path/to/file` 引用关联工作区的文件

### 如何搜索对话内容？

在对话中按 <span class="shortcut">Ctrl</span> / <span class="shortcut">Cmd</span> + <span class="shortcut">F</span> 打开消息搜索面板。

## 会话与工作区

### 多个会话可以同时运行吗？

可以。Polaris 支持最多 16 个会话同时展示，支持 1 行或 2 行网格布局。每个会话独立运行，互不干扰。

### 如何导出对话记录？

鼠标悬停面板顶部的会话标识区域，打开快速切换面板，选择要导出的会话，点击导出按钮。支持导出为 Markdown 或 JSON 格式。

### 工作区之间可以共享文件吗？

可以。通过「关联工作区」功能，AI 可以访问多个工作区的文件。使用 `@{工作区名}:path/to/file` 语法引用关联工作区中的文件。

## 文件与编辑

### 支持哪些语言的语法高亮？

JavaScript, TypeScript, Python, Rust, Go, Java, C++, C, HTML, CSS, JSON, XML, SQL, Markdown, YAML 等 20+ 语言。

### 如何对比文件差异？

在 Git 面板的「变更」标签页中点击任意变更文件，会在 CenterStage 区域打开差异对比视图。

### 编辑器中如何快速搜索文件？

按 <span class="shortcut">Shift</span> + <span class="shortcut">Ctrl</span> + <span class="shortcut">R</span> 打开全局文件搜索面板。

## 定时任务

### 定时任务不执行怎么办？

1. 确认调度器状态为「运行中」（定时任务面板顶部）
2. 确认任务已启用（未被禁用）
3. 检查 Cron 表达式是否正确
4. 查看执行日志是否有错误记录

### 支持哪些 Cron 表达式？

使用标准 5 字段 Cron 语法：`分 时 日 月 星期`

| 表达式 | 含义 |
|--------|------|
| `0 9 * * *` | 每天早上 9 点 |
| `0 9 * * 1-5` | 工作日早上 9 点 |
| `*/30 * * * *` | 每 30 分钟 |
| `0 18 * * 5` | 每周五下午 6 点 |

## 平台集成

### QQ Bot 连接失败？

1. 确认 App ID 和 Client Secret 正确
2. 检查网络连接
3. 尝试使用沙箱环境测试
4. 查看连接状态中的错误类型，对照错误表排查
5. 确认 QQ 开开放平台应用已通过审核

### 飞书机器人如何配置？

1. 前往飞书开放平台创建应用
2. 获取 App ID、App Secret、Verification Token 和 Encrypt Key
3. 在 **设置 → 飞书** 中填入配置
4. 保存并连接

### 可以同时使用 QQ Bot 和飞书吗？

可以。QQ Bot 和飞书使用不同的连接通道，可以同时运行。但同一平台内，同一时间仅一个实例可处于连接状态。

## 性能与故障

### 应用占用内存较大？

- 关闭不需要的会话，减少同时运行的对话数
- 空闲会话会自动释放内存，切换时从快照恢复
- 清理旧的会话历史

### 如何查看应用日志？

打开 **设置 → 高级**，可以看到日志目录路径。在该路径下可以找到应用运行日志文件。

### 如何完全重置应用？

大多数设置页面提供重置按钮。如需完全重置：
1. 关闭 Polaris
2. 删除应用数据目录（在 **设置 → 高级** 中查看路径）
3. 重新启动应用

### 如何报告 Bug？

前往 [GitHub Issues](https://github.com/misxzaiz/Polaris/issues) 提交问题，请包含：
- 操作系统版本
- Polaris 版本号
- 复现步骤
- 错误日志（如有）
